## 一、Hermes RAG（Cornell Tech / Penn State）

### 论文信息

| 字段 | 内容 |
|------|------|
| **标题** | Hermes: Algorithm-System Co-design for Efficient Retrieval-Augmented Generation At-Scale |

| **页码** | 958-973 |
| **DOI** | 10.1145/3695053.3731076 |
| **作者** | Michael Shen, Muhammad Umar, Kiwan Maeng, G. Edward Suh, Udit Gupta |
| **机构** | Cornell Tech / Penn State University |
| **验证状态** | ✅ ACM Digital Library + Semantic Scholar + GitHub三方确认 |

### 关键数据

- 端到端延迟优化 **9.33×**
- 能耗优化 **2.10×**
- 不牺牲检索质量
- 面向万亿级token检索数据存储

### 核心技术

1. **分级检索(Hierarchical Search)**：先在各堆中找最近邻，再全局排序选Top-K1堆，在选中堆内精排Top-K2
2. **DVFS(动态电压频率调节)**：热堆高功耗/高频率，冷堆低功耗/低频率
3. **均衡分堆**：随机1-2%数据试K-means，确保最大堆≤2倍最小堆

| **验证状态** | ✅ arXiv + SJTU官网 + IEEE会议确认 |

### 关键数据

| 模型类型 | 推理加速 | 内存减少 |
|----------|---------|---------|
| BERT/ViT | **4.24×** | **86.7%** |
| GPT系列 | **2.58×** | **90.3%** |

### 核心技术：PipeLoad机制

1. **加载代理(Loading Agent)**：多个并行，从磁盘加载模型层到内存
2. **推理代理(Inference Agent)**：CPU中顺序执行计算
3. **守护代理(Daemon Agent)**：监控内存，及时销毁已完成层的内存

### 框架组成

- **层分析器(Layer Profiler)**：评估每层性能和内存利用率
- **流水线规划器(Pipeline Planner)**：设计最佳执行策略
- **执行引擎(Execution Engine)**：执行PipeLoad机制

## 三、HERMES Math Reasoning（华为 + 香港中文大学）

### 论文信息

| 字段 | 内容 |
|------|------|
| **标题** | HERMES: Towards Efficient and Verifiable Mathematical Reasoning in LLMs |
| **arXiv** | 2511.18760 |
| **DOI** | 10.48550/arXiv.2511.18760 |
| **发布日期** | 2025-11-24 |
| **作者** | Azim Ospanov, Zijin Feng, Jiacheng Sun, Haoli Bai, Xin Shen, Farzan Farnia |
| **机构** | 华为技术 / 香港中文大学 |
| **全称** | Hybrid AgEnt for Reasoning in Mathematics with NEuro-Symbolic Lean4 verification |
| **验证状态** | ✅ arXiv + LessWrong + GitHub三方确认 |

### 关键数据

- AIME'25：最高 **67%** 准确率提升
- 推理FLOPs减少 **80%**
- 四个基准平均准确率提升 **14%**

### 核心架构（四模块）

1. **LLM推理模块**：生成自然语言推理步骤
2. **翻译模块(Autoformalizer)**：将NL步骤转为Lean4代码，反向翻译验证一致性
3. **证明模块(Prover)**：在Lean4中尝试证明/反证
4. **反馈模块**：将验证信号返回LLM

### 关键创新

- **记忆块(Memory Block)**：累积验证过的事实，确保跨步骤一致性
- **中间形式化检查**：防止推理漂移
- **反向翻译**：确保Lean4代码与自然语言含义一致
