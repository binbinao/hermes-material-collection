## 一、项目定位

Hermes Agent 是 Nous Research 开发的**自进化 AI Agent**（Self-Improving AI Agent）。与传统依附于 IDE 的编码副驾或一次性对话式 Chatbot 不同，它被设计为一个**可长期驻留、跨平台运行的自治代理**，核心理念是"运行越久、能力越强"（The agent that grows with you）。

### 核心差异化

| 维度 | 大多数 Agent 框架 | Hermes Agent |
|------|-------------------|--------------|
| 设计目标 | 让 LLM 更好地调用工具 | 让 Agent 越用越聪明 |
| 记忆机制 | 会话结束即失忆 | 持久记忆 + 自动技能沉淀 |
| 学习方式 | 依赖人工配置 | 自创建—自改进—自复用闭环 |
| 部署模式 | 云端 SaaS 或本地单次 | 服务器常驻 + 7种执行后端 |

## 三、Agent 循环深度拆解

### 基本流程

```
接收用户消息 → 构建请求 → 调用 LLM → 解析响应
    ↓                                    ↓
返回最终响应 ← 压缩上下文 ← 执行工具 ← tool_calls?
```

### 核心代码逻辑

```python
# 伪代码
while iterations < max and budget_remaining:
    response = call_llm(messages, tools)
    if response.has_tool_calls:
        for tool_call in response.tool_calls:
            result = execute_tool(tool_call)
            messages.append(tool_result)
    else:
        return response.text
```

### 关键控制机制

#### 迭代预算控制
- **父 Agent**: 默认 90 次迭代
- **子 Agent**: 默认 50 次迭代
- 防止 Agent 陷入死循环

#### 并行工具执行
| 类别 | 示例 | 策略 |
|------|------|------|
| 永不并行 | `clarify`（需用户交互） | 串行 |
| 并行安全 | `web_search`、`read_file`（只读） | 并行 |
| 路径隔离 | `write_file`、`patch` | 条件并行 |
| 最大并发 | — | 8 线程 |

#### 中断与转向
- 不打断正在执行的工具
- 等工具批次完成后处理用户新指令
- 保证操作原子性

#### 多 API 模式
支持 `chat_completions`、`codex_responses`、`anthropic_messages`、`bedrock_converse` 四种模式。

## 四、闭环学习：五大阶段

Hermes 最核心的创新是**自创建—自改进—自复用**的技能循环：

1. **策划记忆** — 任务完成后自主判断值得记住的内容
2. **创建 Skill** — 识别重复模式，自动生成 Markdown 格式 Skill 文件
3. **Skill 自改进** — 现有 Skill 失败时自动优化
4. **FTS5 召回** — 按需检索历史对话（SQLite 全文索引）
5. **用户建模** — 从行为推断偏好

### Curator 后台自学习器

`agent/curator.py` 是差异化核心：
- 后台运行（可配置 `interval_hours`）
- 仅在空闲窗口触发（`min_idle_hours`）
- 自动归档过时技能（`stale_after_days`）
- 模式: 复杂任务 → Curator 提取模式 → 创建技能 → 下次直接命中

## 五、记忆系统：三层架构

### 5.1 情景记忆 — 会话历史
- 当前对话上下文
- 迭代过程中的工具调用与结果

### 5.2 语义记忆 — MEMORY.md / USER.md

**双文件记忆架构**：

| 文件 | 内容 | 字符上限 |
|------|------|----------|
| `MEMORY.md` | 环境事实、项目约定、工具使用技巧 | 2,200 chars |
| `USER.md` | 用户偏好、沟通风格、工作习惯 | 1,375 chars |

**设计哲学**：
- 刻意设置紧凑上限，迫使 Agent 压缩信息、淘汰过期条目
- 记忆满时 `add` 操作失败并返回当前所有条目，促使模型主动决定删除/压缩
- 存储声明性事实（"用户偏好简洁回复"），非命令式指令

**冻结快照模式**：
- 会话开始时将记忆内容作为快照注入系统提示
- 整个会话期间系统提示保持不变
- 中途写入的记忆只更新磁盘文件
- 保持前缀缓存有效，提高 KV cache 命中率

### 5.3 程序性记忆 — Skill 文件

- 存放在 `~/.hermes/skills/` 目录
- 标准 Markdown 格式 (SKILL.md + YAML frontmatter)
- 包含：概述、前置条件、步骤、踩坑记录、示例
- 失败时自动 patch 更新（模糊匹配而非全量重写）

### 5.4 会话搜索
- 基于 SQLite FTS5 全文搜索
- 跨所有会话消息
- 流程: FTS5 召回 → LLM 总结 → 注入上下文

### 5.5 主动记忆与安全扫描

**主动机制**：
- 内置启发式规则 + LLM 辅助判断逻辑
- 自动评估信息是否值得进入长期记忆
- 用户反复提及、明确要求记住、Agent 发现的关键事实 → 优先考虑

**安全扫描**：
- 存储前同步安全扫描，过滤潜在敏感信息（密码、银行卡号等）
- 检测 Prompt Injection 攻击模式

```python
# tools/memory_tool.py 安全模式
_MEMORY_THREAT_PATTERNS = [
    (r'ignore\s+(previous|all|above|prior)\s+instructions', "prompt_injection"),
    (r'do\s+not\s+tell\s+the\s+user', "deception_hide"),
    (r'system\s+prompt\s+override', "sys_prompt_override"),
    (r'curl\s+[^\n]*\$\{?\w*(KEY|TOKEN|SECRET|PASSWORD)', "exfil_curl"),
]
```

## 六、工具系统：自注册 + Toolset 组合

### 工具注册模式

**两阶段暴露**：
1. **自动发现** — 任何 `tools/*.py` 中调用 `registry.register()` 的模块自动导入
2. **手动激活** — 工具必须加入 `toolsets.py::_HERMES_CORE_TOOLS` 才会生效

### 核心工具清单（63个工具）

| 类别 | 工具 |
|------|------|
| Web | `web_search`, `web_extract` |
| 终端 | `terminal`, `process` |
| 文件 | `read_file`, `write_file`, `patch`, `search_files` |
| 视觉 | `vision_analyze`, `image_generate` |
| 技能 | `skills_list`, `skill_view`, `skill_manage` |
| 浏览器 | `browser_navigate` 等 11 个 |
| 规划 | `todo`, `memory` |
| 代码执行 | `execute_code`, `delegate_task` |
| 定时任务 | `cronjob` |

### Toolset 组合系统
- Toolset 之间可互相 `includes`
- 支持递归解析
- 平台特定 toolset 20+ 个
- MCP 支持接入 6,000+ 外部应用

## 七、上下文管理：可插拔压缩引擎

### 压缩策略参数

| 参数 | 默认值 | 作用 |
|------|--------|------|
| `threshold_percent` | 0.75 | 上下文使用率超 75% 触发压缩 |
| `protect_first_n` | 3 | 保护前 3 条消息 |
| `protect_last_n` | 6 | 保护后 6 条消息 |

### 压缩流程
1. 保护头部和尾部消息
2. 中间部分用辅助模型总结
3. 生成结构化摘要（已解决问题、待处理事项、活跃任务）
4. 工具输出裁剪
5. 按比例分配 token 预算

## 八、子 Agent 委托机制

### 隔离设计
- **无父历史** — 子 Agent 看不到父 Agent 对话历史
- **独立终端会话**
- **工具限制** — 禁止特定工具使用

### 被禁止的工具
- 递归委托（防止子 Agent 再生孙 Agent）
- 用户交互类工具
- 记忆写入工具

### 并行与深度控制
| 参数 | 默认值 | 说明 |
|------|--------|------|
| `MAX_DEPTH` | 1 | 扁平结构，最多可配到 3 |
| `MAX_CONCURRENT` | 3 | 最多 3 个并行子 Agent |
| 角色模式 | `leaf` / `orchestrator` | 叶节点或编排者 |

## 九、多平台网关架构

### 支持平台（17+个）

Telegram, Discord, Slack, WhatsApp, Signal, Email, SMS, Home Assistant, Mattermost, Matrix, DingTalk(钉钉), Feishu(飞书), WeChat(微信), WeCom(企业微信), QQ, BlueBubbles, Webhook

### Agent 缓存策略
- 缓存容量: 128 个 Agent 实例
- 空闲淘汰: 1 小时无活动自动清理
- LRU + TTL 淘汰机制

### 网关特性
- 单一进程处理所有平台消息
- 统一处理消息格式差异
- 支持跨平台会话继续（Telegram 开始 → 终端继续）

### 消息存储
- SQLite + WAL 模式（多读者 + 单写者）
- 写竞争处理: 随机抖动重试（15次，20-150ms Jitter）
- 避免护航效应

## 十三、Nudge Engine：何时反思

### 机制
- 两个独立计数器: Memory 计数器(每对话轮次) + Skill 计数器(每工具迭代)
- 达到阈值(默认10)时触发后台审查
- **Fork 独立 Agent 实例**进行审查，输出重定向到 `/dev/null`
- 限制 8 次工具调用，防止 API 浪费
- 审查 Agent 自身 nudge 触发被禁用，防止无限递归

## 十五、架构优势与局限

### 优势
1. **闭环学习是原生能力** — 从架构层面贯通，不是附加功能
2. **工具自注册 + Toolset 组合** — 灵活高效
3. **上下文压缩可插拔** — 留下优化空间
4. **迭代预算 + 中断机制** — 体现深入工程理解
5. **零 CVE 安全记录** — 供应链安全

### 局限
1. 子 Agent 并发上限 3 个
2. 项目迭代节奏快，稳定性需观察
3. Windows 原生不支持（需 WSL2）
4. 生态规模较小（预置技能和平台适配追赶中）
5. 多 Agent 协作不成熟（vs CrewAI/AutoGen 专用框架）
6. Skill 自动生成可能冗余（"太自作聪明"问题）
7. Token 消耗可能高于简单对话式 Agent

## 十六、快速安装

```bash
# 1. 安装
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash

# 2. 配置（交互式向导）
hermes setup

# 3. 开始对话
hermes

# 4. 多平台网关（可选）
hermes gateway setup
hermes gateway

# 5. 从 OpenClaw 迁移
hermes claw migrate  # 一键导入 SOUL.md、MEMORY.md、skills、API keys
```
