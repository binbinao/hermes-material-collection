# Hermes 非技术人员培训大纲

> **适用对象**：产品经理、运营、设计、项目管理、HR、行业架构师、产品架构师（混合编组）
> **培训周期**：6 次，每次 90 分钟
> **核心原则**：零代码、场景驱动、岗位联动、即学即用

---

## 设计理念

现有 `01-培训大纲与资源索引.md` 面向开发者，聚焦部署、架构与代码实战。本大纲面向**非程序员角色**，核心转变：

| 维度 | 开发者版 | 非技术版（本大纲） |
|------|---------|------------------|
| 切入点 | 技术架构 → 功能实现 | 业务痛点 → AI解决方案 |
| 交互方式 | 终端命令行 | WebUI / 企业IM对话 |
| 知识深度 | 源码级理解 | 能力边界认知 |
| 实操形式 | 部署+配置+开发 | 场景模拟+工作流设计 |
| 产出物 | 可运行的Agent | 可落地的AI工作流方案 |

---

## 一、培训目标

完成本培训后，学员将能够：

1. **理解** AI Agent 与聊天机器人的本质区别，建立对 Hermes 能力边界的准确认知
2. **识别** 自己岗位中可被 Hermes 赋能的高价值场景
3. **运用** Prompt 工程技巧与 Hermes 高效协作，获得稳定可靠的输出
4. **设计** 适配自身工作流的 Hermes 集成方案（网关、平台、企业合规）
5. **评估** Hermes 与竞品（OpenClaw 等）的适用场景，做出合理的选型建议
6. **规划** 个人与团队的 AI Agent 落地路线图

---

## 二、课程结构（6次 × 90分钟）

### 第1次：AI Agent 认知升级 —— 从"聊天机器人"到"智能伙伴"

**目标**：破除"AI=ChatGPT"的固有认知，理解 Agent 的自主性与进化性

| 环节 | 时长 | 内容 | 教学方式 |
|------|------|------|---------|
| 开场破冰 | 10min | "你用 AI 做过最满意/最失望的事"——快速收集期望 | 互动问答 |
| 概念建构 | 20min | AI Agent ≠ 聊天机器人：三大核心差异（感知-决策-执行闭环、记忆持久化、技能自进化） | 类比讲解+图示 |
| 生态全览 | 15min | Hermes 五大项目一句话定位：Agent（你的AI伙伴）、4（它的大脑）、RAG（知识检索增强）、Edge（边缘部署）、Math（数学推理验证） | 故事线串联 |
| 实况演示 | 25min | 🔴 三段 Live Demo：<br>① 产品经理：让 Hermes 分析竞品并生成 PRD 初稿<br>② 运营：让 Hermes 从数据中提取周报洞察<br>③ HR：让 Hermes 筛选简历并输出对比表 | 现场演示 |
| 反思总结 | 10min | "Agent 能做什么" vs "Agent 不能做什么"——建立合理预期 | 共识墙 |
| 课后任务 | — | 用 Hermes（或任意AI工具）完成一个工作小任务，记录体验 | 个人实践 |

**关键知识点**：
- Agent 循环：感知 → 思考 → 行动 → 学习（区别于一次性问答）
- Hermes 的"自进化"意味着什么：用得越多越懂你
- 能力边界：创造性判断、情感决策、合规红线仍需人类把关

### 第2次：Hermes 能帮我做什么？—— 六大岗位场景拆解

**目标**：每个学员都能找到 3+ 个与自己日常工作直接相关的 Hermes 应用场景

| 环节 | 时长 | 内容 | 教学方式 |
|------|------|------|---------|
| 场景地图 | 25min | 六大岗位 × Hermes 能力矩阵，逐行拆解（见下方场景表） | 讲解+讨论 |
| 深度演示 | 30min | 选取 3 个跨岗位场景现场演示：<br>① 项目管理：自动追踪进度、生成风险预警<br>② 设计：从需求描述生成设计规范文档<br>③ 行业架构师：分析技术方案并输出对比评估 | 现场演示 |
| 工作坊 | 25min | 分组（按岗位）绘制"我的 Hermes 场景 Top5"，标注优先级和预期收益 | 小组协作 |
| 分享点评 | 10min | 每组 1 分钟闪电分享，讲师点评可行性 | 互动 |

**六大岗位场景矩阵**：

| 岗位 | 高价值场景 | Hermes 能力映射 |
|------|-----------|----------------|
| **产品经理** | 竞品分析、PRD 撰写、需求优先级排序、用户故事生成 | 知识整合 + 结构化输出 + 记忆（积累产品上下文） |
| **运营** | 数据周报生成、活动方案撰写、用户反馈分析、内容策划 | 数据解读 + 模板化输出 + 技能（沉淀运营方法论） |
| **设计** | 设计规范文档、用户画像构建、交互方案评审、设计系统维护 | 结构化思维 + 多轮对话 + 记忆（品牌规范一致性） |
| **项目管理** | 进度追踪、风险预警、会议纪要→行动项、跨团队对齐 | 信息整合 + 定时提醒 + 网关（接入企微/飞书） |
| **HR** | 简历筛选、JD 撰写、培训方案设计、员工满意度分析 | 批量处理 + 模板生成 + 合规审查 |
| **行业/产品架构师** | 技术选型评估、架构方案对比、行业趋势分析、决策支撑 | 深度推理 + 多源验证 + 知识图谱 |

### 第3次：和 AI 高效协作 —— 让 Hermes 真正理解你

**目标**：掌握 Prompt 工程核心技巧，理解记忆与技能系统，从"能用"到"好用"

| 环节 | 时长 | 内容 | 教学方式 |
|------|------|------|---------|
| Prompt 艺术 | 25min | 非程序员的 Prompt 五步法：<br>① 角色设定 → ② 背景交代 → ③ 任务描述 → ④ 输出格式 → ⑤ 约束条件 | 对比演示（差Prompt vs 好Prompt） |
| 记忆系统 | 15min | Hermes 的"长期记忆"：告诉它你的岗位、偏好、常用格式，它下次就不需要重新交代 | 现场演示+体验 |
| 技能进化 | 15min | 你不需要编程——重复做3次的事，Hermes 自动沉淀为技能；下次一句话触发 | 案例讲解 |
| 实操练习 | 25min | 学员用五步法为自己的 Top1 场景撰写 Prompt，现场测试并迭代 | 个人实操 |
| 互评优化 | 10min | 同组互评 Prompt，讲师选取典型优化案例 | 互动 |

**Prompt 五步法示例**：

```
❌ 差的 Prompt：
"帮我写个竞品分析"

✅ 好的 Prompt（五步法）：
"你是一位资深产品经理（①角色），我们正在做一款面向B端的AI Agent平台（②背景），
请对 Hermes Agent 和 OpenClaw 进行竞品分析（③任务），
输出为对比表格，维度包括定位、核心能力、适用场景、局限性（④格式），
分析需基于公开信息，不确定的标注'待验证'（⑤约束）"
```

### 第4次：Hermes 接入我的工作流 —— 网关与平台集成

**目标**：理解如何将 Hermes 连接到已有的企业工具链，无需编程即可实现自动化

| 环节 | 时长 | 内容 | 教学方式 |
|------|------|------|---------|
| 网关概念 | 15min | "网关 = Hermes 的手机号"——让 Hermes 能接收和发送消息到企微/飞书/邮件等 | 类比讲解 |
| 支持平台 | 10min | 17+ 平台一览：企业微信、飞书、钉钉、Slack、Discord、Telegram、邮件等 | 图表展示 |
| WebUI 操作 | 20min | 通过 RDSHermes 企业版 WebUI 完成网关配置（全程界面操作，零终端命令） | 现场演示 |
| 安全与合规 | 10min | 数据不出域、凭证加密、审计日志——腾讯内部使用须知 | 要点讲解 |
| 工作坊 | 25min | 分组设计"我的 Hermes 工作流"：<br>· 选择 2 个接入平台<br>· 绘制信息流转图<br>· 标注触发条件与预期输出 | 小组协作 |
| 展示交流 | 10min | 每组展示工作流设计 | 互动 |

**典型工作流示例**：

```
项目经理的每日站会自动化：

  企微群消息 → Hermes 网关 → 自动收集昨日进展
                                    ↓
                              整合为站会摘要
                                    ↓
  企微群 ← Hermes 网关 ← 推送站会纪要 + 行动项
```

### 第5次：AI Agent 选型与评估 —— 决策者视角

**目标**：能够基于业务需求做出 Agent 选型判断，理解 Hermes 与竞品的差异定位

| 环节 | 时长 | 内容 | 教学方式 |
|------|------|------|---------|
| 选型框架 | 15min | AI Agent 选型五维评估模型：能力、易用性、生态、安全、成本 | 框架讲解 |
| 对比分析 | 20min | Hermes vs OpenClaw 核心差异（非技术语言）：<br>· Hermes = 长期进化的伙伴（越用越懂你）<br>· OpenClaw = 开箱即用的工具（多平台万能钥匙）<br>· 二者可互补，非互斥 | 案例对比 |
| 企业部署 | 15min | 三种部署方式对比：<br>① 开源版自托管（需技术团队支持）<br>② RDSHermes 企业版（零配置WebUI）<br>③ 云端 API 接入（轻量集成） | 方案对比 |
| 架构师专场 | 20min | 面向行业/产品架构师的深入议题：<br>· 技能治理策略（避免技能冗余与冲突）<br>· 多 Agent 协作架构<br>· 与腾讯云现有服务的集成切面 | 深度讲解 |
| 决策演练 | 15min | 给定 3 个业务场景，小组讨论选择哪个 Agent + 部署方式，说明理由 | 小组讨论 |
| 总结 | 5min | 选型没有标准答案，只有最适答案 | 点评 |

**选型五维评估模型**：

| 维度 | 关键问题 | 权重建议 |
|------|---------|---------|
| **能力** | 是否覆盖核心场景？能力边界在哪？ | ★★★★★ |
| **易用性** | 非技术人员能否独立使用？学习曲线多陡？ | ★★★★ |
| **生态** | 能否接入现有工具链？社区活跃度？ | ★★★ |
| **安全** | 数据是否合规？审计能力是否完备？ | ★★★★★ |
| **成本** | 部署成本、运维成本、Token消耗？ | ★★★ |

### 第6次：实战演练 —— 打造你的 AI 工作流方案

**目标**：每人/每组产出一个可在培训后立即落地的 Hermes 工作流方案

| 环节 | 时长 | 内容 | 教学方式 |
|------|------|------|---------|
| 方案框架 | 10min | "AI 工作流方案"标准模板讲解：场景→痛点→方案→预期收益→落地步骤 | 讲解 |
| 分组实战 | 35min | 按岗位分组，每组完成一个完整方案：<br>· 填写方案模板<br>· 在 Hermes 中实际测试核心 Prompt<br>· 绘制工作流示意图 | 小组协作 |
| 方案路演 | 25min | 每组 5 分钟路演（3分钟展示 + 2分钟Q&A） | 路演 |
| 评审颁奖 | 10min | 评选"最佳落地奖""最具创意奖""最跨岗协同奖" | 互动 |
| 路线图 | 10min | 个人 30 天落地路线图 + 持续学习资源 | 讲解 |

**AI 工作流方案模板**：

```markdown

### 基本信息
- 团队成员：
- 岗位：
- 方案名称：

### 场景描述

- 期望效果：（节省X小时/周，准确率提升至Y%）

### Hermes 方案
- 使用的 Hermes 能力：（对话/记忆/技能/网关/多平台）
- 核心 Prompt：（用第3次课的五步法撰写）
- 接入平台：（企微/飞书/邮件/其他）
- 工作流示意图：

### 落地计划
```

## 三、教学设计说明

### 3.1 混合编组策略

```
推荐分组方式：异质分组（每组包含不同岗位）

组A：PM + 运营 + 架构师    → 偏"产品落地"场景
组B：设计 + 项目管理 + HR  → 偏"流程优化"场景
组C：架构师 + PM + HR      → 偏"企业级部署"场景

第6次实战时允许自由组合，鼓励跨岗协作
```

### 3.2 渐进式能力建构

```
第1次    认知层    "AI Agent 是什么"
第2次    识别层    "跟我有什么关系"
第3次    操作层    "我该怎么跟它说话"
第4次    集成层    "怎么接入我的工具"
第5次    决策层    "该不该用、怎么选"
第6次    创造层    "设计我自己的方案"
```

### 3.3 与技术版培训的关系

| 本大纲（非技术版） | 技术版（01-培训大纲） | 衔接建议 |
|---|---|---|
| 第1次：Agent认知 | M1+M2：架构概览 | 非技术版先上，建立需求认知 |
| 第2次：岗位场景 | —（技术版无此内容） | 非技术版的场景清单可指导技术版的部署优先级 |
| 第3次：Prompt+记忆+技能 | M4：记忆与技能系统深度 | 同一知识点，不同深度 |
| 第4次：网关集成 | M5：多平台网关与安全 | 非技术版重操作，技术版重原理 |
| 第5次：选型评估 | M7：竞品对比 | 非技术版重决策框架，技术版重技术对比 |
| 第6次：实战路演 | M9：企业落地 | 非技术版出方案，技术版出部署 |

**推荐排期**：非技术版 Day 1-2 → 技术版 Day 3-4，形成"业务先明确需求，技术再落地实现"的闭环。

## 六、培训效果评估

### 过程评估（每次课后）
- 课堂互动参与度
- 实操练习完成质量
- 课后任务提交率

### 结果评估（培训结束时）
- 第6次路演方案质量评分（满分5分）
- 培训满意度问卷
- 30天后回访：方案落地执行率

### 评估维度

| 维度 | 合格 | 良好 | 优秀 |
|------|------|------|------|
| 场景识别 | 能说出2个场景 | 能匹配5个场景 | 能设计跨岗位场景 |
| Prompt 质量 | 五步法基本完整 | 输出稳定可用 | 可沉淀为技能复用 |
| 方案设计 | 有框架 | 有落地计划 | 有量化预期收益 |
| 选型判断 | 知道二者区别 | 能给出选型理由 | 能结合业务做决策 |
