# "龙虾"过时了，现在流行"养马"

---

## 一、核心现象

"养马"现象指 Hermes Agent 走红后引发的新一轮"新技术—新焦虑—新生意"循环。

### 1. "马"的定义

Hermes 是 Nous Research 团队于 2026年2月25日推出的开源 Agent 项目，主打**自我进化能力**，区别于依赖用户操作的 OpenClaw，因此快速走红。

### 2. 走红

2026年4月后，Hermes 整体日均 Token 消耗量从 20 亿激增至 3000 亿，冲进 OpenRouter 等多个开发者平台榜单前列。

### 3. 衍生生意

社交媒体、电商平台已出现 39.99 元"专业安装"服务、"从入门到精通"教程等"养马"相关生意，对应此前 OpenClaw 流行时的"养龙虾"配套服务逻辑。

---

## 二、Hermes 比 OpenClaw 的技术优势

Hermes 的核心突破是重构了 Agent 的学习与记忆机制，实现从"工具属性"向"自主进化"的转变。

| 对比维度 | OpenClaw | Hermes |
|----------|----------|--------|
| **技能学习** | 依赖用户手动构建、整理技能与记忆，本质是高度可编排的工具集合 | 采用**可写运行时架构**，运行中可自动生成、优化、存储新技能代码，通过"技能蒸馏"机制将任务经验沉淀为可复用技能文件，不依赖人工干预；完成任务后会自动保存技能，下次遇到相似任务可直接调用并迭代 |
| **记忆机制** | 全量被动记录，存储所有对话上下文、用户偏好，信息完整可追溯，但长期使用后数据噪声多、Token 消耗高、检索精度和响应速度下降 | 选择性主动记忆，对话中高频触发回顾机制，提炼值得写入长期记忆的信息；引入复杂记忆后端，抽象提取用户偏好、行为模式等高层级信息，可提升记忆调用效率 |

### 专家与用户实测反馈

- **奇安信 AI 安全专家刘岩**：Hermes 不是执行预设指令，而是"自己在编写自己的能力"。
- **AI 博主人工大黑**：Hermes 执行任务时会主动想办法完成，且长期使用后 Token 消耗量比 OpenClaw 更低。
- **深度科技研究院院长张孝荣**：Hermes 代表了 Agent 从"任务执行"向"认知规划"的进化方向，带记忆的自主智能体是未来成熟 Agent 的必经路径。

---

## 三、Hermes 当前存在的局限性

尽管技术方向先进，但 Hermes 远未成熟，实际体验与理想预期存在明显差距：

1. **版本仍处于实验阶段**：最新版本为 0.1，属于初阶实验性产品，和 OpenClaw 没有质变级的差异。

2. **技能蒸馏功能冗余**：自动生成技能的逻辑过度，会生成大量无意义的冗余技能，不仅没有提效，反而增加管理成本，"自动进化"反而成了使用负担。

3. **存在安全隐患**：自进化过程的不确定性导致风险更高，Agent 可能从恶意邮件、含隐藏指令的网页、被投毒的文档等外部数据中自主学习到危险行为，且恶意行为是自我演化产生的，难以提前防御。

4. **记忆机制仍不完善**：复杂任务中过度抽象记忆可能丢失关键细节，长期使用后记忆体系可能出现结构混乱，仍存在"忘事"的问题。

5. **使用门槛高**：仍依赖服务器部署和环境配置，非技术用户从安装、调试到日常维护都有较高难度，远未实现开箱即用。

6. **用户使用认知误区**：多数用户追新是被 FOMO（错失恐惧）情绪驱动，属于"数字囤积"类的情绪消费，而非基于真实需求；若没有明确使用场景就盲目使用，不仅无法提升效率，还会浪费大量 Token、增加使用成本。

---

## 四、给普通用户的建议

多位专家提示：对于普通用户而言，Hermes 还不是需要立刻投入时间、成本深度使用的工具。如果连 OpenClaw（"龙虾"）都没使用明白，现阶段不需要跟风尝试 Hermes。

---

## 五、行业意义

"养马"现象的火爆，本质是 Hermes 这款开源 Agent 项目引发的新一轮技术焦虑与商业机会的叠加。它既反映了 AI Agent 技术的快速迭代，也提醒用户在追新时需要基于真实需求，而非被 FOMO 情绪驱动。

Hermes 代表了 Agent 从"任务执行"向"认知规划"的进化方向，但当前版本仍不成熟，存在技能冗余、安全隐患、记忆不完善、使用门槛高等问题。对于普通用户，专家建议先掌握现有工具，再逐步尝试新平台。
